# Микросервис уведомлений (Notification Service)

[![CI](https://github.com/bugfix666/notify-service/actions/workflows/ci.yml/badge.svg)](https://github.com/bugfix666/notify-service/actions/workflows/ci.yml)

REST API-сервис на Laravel 12 (PHP 8.4): массовая рассылка SMS/Email с приоритезацией, надёжной доставкой и идемпотентностью

Сервис построен по сервис‑репозиторийной архитектуре, использует общую инфраструктуру (PostgreSQL, RabbitMQ, Redis) и полностью упакован в Docker

---

## Функциональность

- Массовая отправка SMS или Email через REST API
- Приоритезация: транзакционные сообщения (коды доступа и тп) обрабатываются вне очереди, обгоняя маркетинговые
- Статусы доставки: `queued` => `delivered` / `failed`
- Гарантии доставки:
  - at‑least‑once через RabbitMQ
  - exactly‑once на уровне бизнес‑логики (дедубликация по `idempotency_key`)
  - Повторные попытки при временной недоступности шлюза (retry с экспоненциальной задержкой)
- История уведомлений по пользователю

## Архитектура и ключевые паттерны

- **Сервис‑репозиторный слой** - бизнес‑логика вынесена в сервисы, доступ к данным через интерфейсы и репозитории
- **Очереди с приоритетами** - отдельные очереди `high` (транзакционные) и `default` (маркетинговые), воркер слушает в порядке `high,default`
- **Идемпотентность** - ключ идемпотентности проверяется перед созданием записей, дополнительно используется Redis‑блокировка для предотвращения гонок
- **Шлюзы‑заглушки** - `SmsGateway` и `EmailGateway` имитируют отправку/логирование, легко заменяются на реальные интеграции
- **API‑ресурсы Laravel** - единый формат ответов (JSON)
- **Строгая типизация и перечисления (enum)**

## Быстрый старт

1. Клонируйте репозиторий:
   ```bash
   git clone git@github.com:bugfix666/notify-service.git
   cd notify-service
   ```

## Приоритезация и надёжность

- **Приоритеты:** Транзакционные сообщения (ключ `priority=transactional`) попадают в очередь `high` и обрабатываются воркером немедленно, обгоняя маркетинговые (`priority=marketing`, очередь `default`)
- **Надёжность:**
    - Сообщения сохраняются в RabbitMQ (персистентность)
    - При ошибке шлюза джоба повторяется до 3 раз с задержками 10/30/60 секунд
    - Дедубликация по ключу идемпотентности гарантирует, что повторный запрос не создаст дубликатов

## Примеры запросов

### Отправка массового уведомления
```bash
curl -X POST http://localhost/api/v1/notifications/send \
  -H "Content-Type: application/json" \
  -d '{
    "channel": "sms",
    "text": "Ваш код: 1234",
    "user_ids": [1, 2, 3],
    "priority": "transactional"
  }'
```

### История уведомлений пользователя
```bash
curl http://localhost/api/v1/users/1/notifications
```
